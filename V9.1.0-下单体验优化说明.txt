红祺腾达官网 V9.1.0｜客户下单体验优化版

一、保持不变
1. 官网主页 index.html 的原有布局、内容、图片和导航不变。
2. AI项目、计算模拟、分析表征、耗材仪器的项目详情内容不变。
3. 官网继续使用现有 CloudBase webPortal 接口，与小程序共用客户账户和业务数据。

二、下单流程简化
原流程：项目页 → 客户中心 → 多栏目门户 → 选择表单 → 填写大量专业字段。
新流程：项目详情页 → 填写项目需求 → 常用选项 → 一句话描述 → 提交并生成 Word。

三、项目详情页
1. 仅在具体项目详情页底部增加简洁固定栏。
2. 显示当前项目名称。
3. 主按钮为“填写项目需求”。
4. 自动携带项目编号、项目名称和业务类型进入客户中心。
5. 不再在官网主页和所有项目列表卡片中批量增加“立即下单”按钮，避免页面杂乱。

四、需求填写页
1. 从项目详情进入后，项目自动选中，无需重复查找。
2. 只显示 3–4 个常用选项：
   - AI项目：主要目标、现有数据、希望获得的成果。
   - 计算模拟：研究体系、已有文件、重点结果。
   - 分析表征：样品数量、样品状态、安全情况、数据分析需求。
   - 耗材仪器：数量、规格型号、使用场景。
3. 核心必填项仅保留：需求描述、联系人、手机号或邮箱。
4. 提供“使用填写提示”和“请技术人员评估”，客户不知道参数时可以直接生成描述框架。
5. AI项目、计算模拟、分析表征及耗材采购的专业 Word 模板放在“专业模板（备选）”中，默认折叠。
6. 支持在线填写专业模板，也支持下载对应 Word 模板。
7. 支持上传 Word、PDF、Excel、图片、压缩包和常用结构文件。
8. 自动在浏览器保存未提交草稿。
9. 提交后可直接生成并下载 Word 需求单。

五、客户中心
1. 保留与小程序一致的首页、提交需求、业务进度、交付文件、售后、资金账户、合同、发票、消息、成员和资料设置。
2. 修复 V9.0.0 中客户中心 HTML 与 portal.js 结构不一致的问题。
3. 报价、项目、交付和售后按卡片显示，移动端更易查看。
4. 支持报价确认和申请改价接口。
5. 支持多账户切换接口。

六、部署
将压缩包解压后，把全部文件覆盖上传到当前 GitHub 官网仓库根目录。不要只上传 customer-portal 文件夹。
CloudBase webPortal 云函数继续使用 V8.0.5，无需重新填写环境变量。
